1. PPT 슬라이드 마스터
	PPT 전체 디자인을 한 번에 관리하는 원본 툴

1) 슬라이드 마스터
	여러 슬라이드에 공통으로 적용되는 디자인, 글꼴, 색상, 로고,
	제목 위치, 페이지 번호 관리할 수 있는 기능

2) 사용하는 이유
	전체 디자인을 통일할 수 있음
	수정이 빠름
	실무에서 많이 사용함

3) 슬라이드 마스터
	보기 탭 → 슬라이드 마스터

4) 슬라이드 마스터 화면 구조
	가장 위에 있는 큰 슬라이드 → 전체 마스터 슬라이드
		전체 글꼴, 전체 배경, 공통 로고, 공통 하단 문구, 페이지 번호 위치
	아래쪽에 있는 작은 슬라이드 → 레이아웃 슬라이드
		슬라이드의 용도가 다름
		슬라이드 종류별 세부 설정(제목슬라이드, 제목 및 내용 슬라이드, 섹션 제목 슬라이드, 두개 콘텐츠 슬라이드, 비교 슬라이드, 빈 화면 슬라이드, 그림 포함 슬라이드 등)

5) 슬라이드 마스터에서 할 수 있는 것
- 글꼴 설정
- 배경 설정
- 로고 삽입
	슬라이드 마스터 → 삽입 →  그림 → 로고 선택 → 왼쪽 상단이나 오른쪽 상단/하단에 배치
- 페이지 번호 설정
	삽입 → 슬라이드 번호 → 슬라이드 번호 체크 → 모두 적용
- 하단 문구 넣기
- 제목 위치 고정
	x 위치 : 왼쪽 1cm
	y 위치 : 위쪽 0.8cm
	글자 크기 : 30pt
	색상 : 진한 남색

6) 자리 표시자
	내용을 넣을 수 있는 미리 정해진 박스
	제목, 본문, 그림, 차트, 표, SmartArt, 미디어, 날짜, 슬라이드 번호, 바닥글
	자리 표시자의 위치와 크기 미리 지정 가능

7) 슬라이드 마스터와 일반 편집 차이
			일반 슬라이드 편집		슬라이드 마스터 편집
	목적		현재 슬라이드 내용 수정	전체 양식 수정
	적용 범위		해당 슬라이드만		여러 슬라이드에 적용
	사용 시점		발표 내용 작성		디자인 틀 만들기

3. 상세보고서
	6개 엑셀파일
	- 네트워크, Linux Server, Windows Server, DBMS, PC, 보안장비

1) hidden 시트 역할
	해당 점검표에서 정한 내부 위험평가 기준표(모든 기관에서 공통으로 쓰는 절대 공식이 아님)
	이 시트를 기준으로 위험도를 산정한다

	실제 점검 내용을 입력하는 곳이 아닌 다른 시트에서 계산할 때 참고하는 기준표
	보통 hidden 시트로 숨겨놓고 사용자는 점검표 시트에서만 입력하도록 구성됨

	중요도 : H
	점검결과 : N
	자산 등급 : H
	위협도 : H
	보호대책 : N

	이 값을 바로 계산하지 않고 hidden 시트의 기준표를 보고 숫자로 바꾼다
	H => 3점
	N => 위험평가 3점
	최종 위험도 => 평가표에서 교차값 조회

2) 전체 흐름
	점검 항목 입력
	중요도, 자산 등급, 위협도 입력
	점검 결과 입력	Y N N/A 등을 입력
	hidden 시트 기준으로 점수 변환
	위험도 평가표에서 위험점수 산정
	위험 점수 범위에 따라 H/M/L 등급 결정
	
	문자 입력 -> 숫자 변환 -> 위험점수 계산 -> 위험 등급 산정(DoA)

3) 점검결과 점수
취약점 진단 => 얼마나 잘 지켰는가를 보는 점수
   	점검 이행률 계산시 사용
   	Y = 1
   	N = 0
   	P = 0.5   부분 충족
   	N/A = 0
   	R/A = 0
   	Y(즉시조치) = 1

   	점검항목이 10개라고 가정
   	Y 7개
	N 3개

   	Y = 7 X 1 = 7점
   	N = 3 X 0 = 0점
   	점검 이행률 7 / 10 = 70%

위험평가 => 얼마나 위험한가를 보는 점수
   	Y = 0
   	N = 3
   	P = 0.5
   	N/A = 0
   	R/A = 0
   	Y(즉시조치) = 0


4) DoA   위험 수용 기준선(고위험 판단 기준값)
	최종 위험도 점수 범위
	H = 21 ~ 30
	M = 11 ~ 20
	L = 1  ~ 10
	DoA가 21이라는 것은 위험점수가 21점 이상이면 고위험으로 본다
	=> 우선 조치 대상으로 분류

5) 자산 등급 점수 범위
	위험등급	최저	최고
	H	7	9
	M	5	6
	L	3	4

	자산의 중요도를 평가해서 결정하며 기밀성, 무결성, 가용성을 각각 H/M/L로 평가하고 점수를 바꿈
	EX) 기밀성 H, 무결성 H, 가용성 H
		총점 : 9	자산등급 : H
	EX) 기밀성 L, 무결성 M, 가용성 H
		총점 : 6	자산등급 : M

	자산등급은 해당 장비나 시스템이 얼마나 중요한지를 나타냄
	같은 취약점이라도 중요 서버에서 발견되면 위험도가 높아지고, 테스트 PC에서 발견되면 상대적으로 낮아질 수 있음

6) 위협도 점수 범위
	위협도는 공격 가능성이나 노출 수준을 평가하는 값
	외부 노출 여부, 공격 가능성, 악용 난이도, 공격 빈도, 접근 가능성 등의 요소로 평가
	
	공격 가능성 H, 노출 수준 H
	총점 : 6	위협도 : H

	취약점 자체보다는 실제 공격이 발생할 가능성을 보는 값
	외부에 노출되어있고 공격자가 쉽게 접근 할 수 있으면 위협도는 높아짐

7) 위험도 점수 범위
	최종 위험 점수를 H/M/L로 바꾸는 기준
	위험점수	30 => H
	위험점수	20 => M
	위험점수	1 => L

8) 위험도 평가표
	Y = 0 보호대책 있음
	이미 위험을 줄이는 조치가 있다는 의미
	방화벽 접근제어 있음
	ACL 설정 있음
	관리자 접근 IP 제한 있음
	MFA 적용됨
	로그 모니터링 적용됨
	취약 서비스는 비활성화 됨
	=> 취약점이나 위협 요소가 있더라도 보호대책이 이미 적용되어있으면 실제 위험은 줄어들게 됨
	보호대책 있음 영역은 위험점수는 낮게 산정됨

	Y = 1 보호대책 없음
	위험을 줄이는 장치가 없다는 의미
	외부 접근 제한 없음
	방화벽 정책 없음
	관리자 접속 제한 없음
	패스워드 정책 없음
	취약 서비스 그대로 사용

- 위험도 평가표 읽는 순서
	보호대책 있는지 확인한다
	보호대책 있음이면 Y=0 영역을 본다
	보호대책 없음이면 Y=1 영역을 본다
	취약점 등급 L/M/H를 찾는다
	위협도 등급 L/M/H를 찾는다
	자산 등급 L/M/H 행을 찾는다
	교차되는 값을 위험점수로 사용한다
	위험점수 범위에 따라서 H/M/L 등급을 정한다

	보호대책 -> 취약점 -> 위협도 -> 자산등급 -> 위험점수

4. 위험도
	위험도 = 자산의 중요도 x 취약점과 위협의 영향
	R = A x (1 + V' x T) = A x (1 + (V x C) x (F + S))

	약자	영어			한글의미
	R	Risk			위험도
	A	Asset Criticality	자산의 중요도
	V	Vulnerability		취약점
	V'	Adjusted Vulnerability	보정된 취약점(보호대책을 반영한 취약점) => V' = V x C
	C	Control/Countermeasure	보호 대책
	T	Treat			위협도		=> T = F + S
	F	Frequency		발생 빈도
	S	Severity		심각도

=> 중요한 자산일수록 위험도가 높아진다
=> 보호대책이 있으면 취약점의 영향이 줄어든다
=> 발생빈도와 심각도가 높으면 실제 위험도도 커진다
