제안요청서(RFP)
	이런 사업을 하고 싶은데 어떻게 할건지 제안서 제출해줘

1) 현재 상황
- 웹 서비스 구축 완료된 상황
	인프라 구축 필요
	웹 모의해킹 필요
	SIEM 구축 필요
	를 하기 위해 RFP를 배포

2) RFP에 들어가는 내용
- 사업 개요
	사업명, 사업 목적, 사업 기간, 사업 예산
- 현재 환경
	서버 환경, 네트워크 환경, 보안장비 환경
- 요구 사항
	서버 구축
	네트워크 구축
	WAF 구축
	웹 모의해킹
	SIEM 구축
- 산출물
	WBS, 구성도, 점검 보고서, 최종 보고서

2. 제안서
	RFP를 받은 업체가 우리는 이렇게 구축할게 라고 작성하는 문서

1) 관계
	고객 → RFP → 업체 → 제안서 → 고객

2) 제안서 들어가는 내용
- 사업 이해
	고객 요구사항 분석
	현재 문제점 분석

- 수행 방안
	구축 방법
	점검 방법
	운영 방법

- 일정
	WBS

- 조직
	PM
	네트워크
	서버
	보안
	모의해킹

- 기대효과
	보안성 향상
	서비스 안정성 향상

3. WBS(Work Breakdown Structure)
	프로젝트를 작은 업무 단위로 나눈 일정표

	프로젝트 진행률 관리
- 요구사항 분석	1일
- 네트워크 설계	3일
- 서버 구축/네트워크 구축/보안 장비 구축	10일
- 인프라 취약점 진단	
- 모의해킹		4일
- SIEM 구축	4일
- 테스트		1일
- 최종 보고서

4. 네트워크 구성도
	시스템이 어떻게 연결되는지 표현한 그림

1) 필요한 이유
	어던 장비가 있는지, 어디에 위치하는지, 서로 어떻게 통신하는지, 어떤 보안장비를 거치는지
	한 눈에 확인할 수 있도록 만든 그림

- 구축 기준
	어떤 장비를 어디에 설치하고 어떻게 연결할지 기준이 됨
	Firewall 뒤에 WAF 설치
	WAF 뒤에 WEB 서버
	WEB 서버와 WAS 서버와 연결
	WAS 서버와 DB 서버와 연결

	개발팀, 운영팀, 기획팀, 재무팀, 영업팀

- 보안 설계
	외부에서 내부 서버로 직접 접근이 가능한지, 보안장비가 적절하게 배치되었는지 확인

2) 확인 가능 항목
	서버 위치
	네트워크 구간
	통신 경로
	보안장비 배치
	망분리 구조(DMZ, 내부망, 외부망)

3) 포함되어야하는 내용
	인터넷
	방화벽
	WAF
	VPN
	WEB
	WAS
	DB

※ 구성도 그릴 때 주의할 점
1. 무엇을 구축할 지 먼저 생각한다
	WEB 서버, WAS 서버, DB 서버
	DNS 서버, DHCP 서버, MAIL 서버
	Log 서버, SIEM

2. 어디에 배치할 것인지 생각한다
	서비스 특성에 따라 망 분리를 해야한다
	인터넷에서 직접 접근해야하는 서버는 DMZ망에 두고 중요한 서버는 내부망에 둔다

	DMZ, 내부망, 백업망

3. 어떻게 보호할 것인지 생각한다
	Firewall 네트워크 접근 제어
	WAF	웹 공격 차단
	VPN	본사-지사 암호화 통신

4. 어떻게 탐지할 것인지 생각한다
	Log Server, SIEM
